# Replication Code for: "A Joint Model for Multivariate Longitudinal Biomarkers and Survival Outcomes with a Fast Kalman Filter"

**Authors:** Mingzhao Hu, Ya Luo, Yuedong Wang  

---

## Overview

This repository contains code for two parts of the paper:

1. **Real-data analysis**
   - joint model fitting,
   - bootstrap confidence intervals,
   - real-data trajectory and prediction figures.

2. **Simulation and computational analysis**
   - filtering-time comparison,
   - simulation MSE, bias, variance, and bootstrap coverage calculations.

The currently uploaded public files are sufficient for reproducing the simulation results in **Table S2** and **Table S3**. The real-data scripts are included, but the restricted real-data RData files and several precomputed real-data fit objects are not currently included in the public repository.

## Simulation and Computational Analysis

The simulation and computational scripts reproduce the appendix simulation results.

---

### Table S2: Filtering-Time Comparison

Run:

```r
source("filter_time_comparison.R")
```

This script compares the filtering time of the proposed fast Kalman filtering algorithm with the univariate-treatment approach.

It uses:

```text
llh_OU.R
llh_OU_unitrt.R
```

and generates the timing results corresponding to **Table S2**.

Depending on the version of the script, outputs may be saved as:

```text
filter_time.Rdata
filter_time_unitrt.Rdata
```

or under a `filter_time_results/` folder.

---

### Table S3: Simulation MSE, Bias, Variance, and Bootstrap Coverage

Run:

```r
source("sim_bootstrap_analysis.R")
```

This script reads:

```text
theta_est_mat.Rdata
true_val.Rdata
theta_mat_est_extra_final.Rdata
```

and computes:

```text
MSE
Variance
Bias
Bootstrap coverage
```

These results correspond to **Table S3**.

The key archived bootstrap object is:

```text
theta_mat_est_extra_final.Rdata
```

This object is the final set of 100 bootstrap simulation objects used for the reported bootstrap coverage calculation.

The script saves or prints:

```text
par_est_df.Rdata
par_est_boot_df.Rdata
```

where `par_est_boot_df.Rdata` is generated when the script is run.

## Bootstrap Simulation Files

### Exact bootstrap object used for Table S3

For exact replication of Table S3, use:

```text
theta_mat_est_extra_final.Rdata
```

This object contains the final archived bootstrap estimates used in the paper. It should be loaded directly by:

```text
sim_bootstrap_analysis.R
```

A fresh run of `sim_bootstrap_gen.R` should not be expected to recreate the exact same historical bootstrap object bit-for-bit.

---

### `sim_bootstrap_collect.R`

This script collects 100 individual bootstrap estimate files into one list object.

It expects files such as:

```text
bootstrap_results/results_m1000/theta_est_mat1.Rdata
bootstrap_results/results_m1000/theta_est_mat2.Rdata
...
bootstrap_results/results_m1000/theta_est_mat100.Rdata
```

and saves:

```text
theta_mat_est_extra_final.Rdata
```

This step is optional if `theta_mat_est_extra_final.Rdata` is already uploaded.

At present, the repository contains the folder:

```text
bootstrap_results/results_m1000/
```

but users should check whether the 100 individual `theta_est_mat*.Rdata` files are actually present inside that folder before running `sim_bootstrap_collect.R`.

---

### `sim_bootstrap_gen.R`

This script documents the standardized bootstrap-generation procedure:

1. selecting the simulation dataset used for bootstrap resampling,
2. resampling subjects with replacement,
3. reconstructing alive-at-time and present-at-time subject lists,
4. fitting the joint model by likelihood optimization,
5. transforming log-scale parameters back to the natural scale,
6. saving bootstrap estimate matrices.

This script is included to show the bootstrap-generation procedure.

However, the reported Table S3 should be reproduced from:

```text
theta_mat_est_extra_final.Rdata
```

rather than by rerunning `sim_bootstrap_gen.R`.

A full stochastic rerun of `sim_bootstrap_gen.R` requires additional simulation objects that are not currently uploaded, including:

```text
X00_cube_in_list.Rdata
MSE_fix_mat.Rdata
```

The file `X00_cube_in_list.Rdata` may be too large for direct upload through the GitHub web interface. It is not required for exact replication of Table S3 if `theta_mat_est_extra_final.Rdata` is used.

## Reproducibility Notes

1. Run scripts from the repository root unless paths are changed manually inside the scripts.

2. File extensions are case-sensitive on Mac and Linux. For example, `.RData` and `.Rdata` are different.

3. For exact simulation/bootstrap replication, use:

```text
theta_mat_est_extra_final.Rdata
```

4. `sim_bootstrap_gen.R` documents the standardized bootstrap-generation procedure. It is not expected to recreate the exact historical 100 bootstrap objects used for Table S3.

5. Real-data outputs require restricted or precomputed RData files that are not currently included in the public repository.

6. The filtering-time comparison may take substantial time for large sample sizes.
